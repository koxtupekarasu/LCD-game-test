# LCDゲーム仮 正本ドキュメント：現行実装仕様

最終整理: 2026-04-04

この文書は、現時点で**実装済み・現行として扱う仕様のみ**をまとめた正本である。旧案・保留・要調整は別冊の「未実装・保留メモ」を参照。

---

## 1. 全体方針
- 作品の基本形は**携帯育成液晶ゲーム風**。
- 描画は **Canvas方式** を基本とし、一部情報表示は **DOM overlay** を使う。
- 画面内の用語や設計は、単なるRPGではなく**観測・管理・干渉**を伴う育成機として整理する。
- ただしUI表層の見え方は、まず**育成玩具として一目で分かること**を優先する。
- 内部論理解像度は **384 × 288**。
- モンスター表現の基本単位は **16×16 / 1bitドット**。
- HiDPI対応あり。CSS表示サイズは維持しつつ、canvas実解像度は devicePixelRatio で高解像度化する。

## 2. 基礎アプリ構成
- 基本ファイル:
  - `index.html`
  - `app.js`
  - `style.css`
  - `manifest.webmanifest`
  - `service-worker.js`
- 静的配信で動作確認済み。
- 実時間差分ベースの更新を採用しており、固定 `1/60` 更新は廃止済み。
- `performance.now()` ベースの `dtSec` を使い、`0〜0.1` にクランプする。

## 3. 世界内の基本存在（実装前提の用語）
- 媒体の正式名称: **AD（Artifact Disk / アーティファクト・ディスク）**
- 個体群の正式英名: **Artifact Domain**
- 学術和名: **AD由来疑似生命体領域群**
- ゲーム内通称: **アド**
- ゲーム内では、個体群も単体個体も基本的に「アド」または個体名で呼ぶ。
- `AD` は媒体そのもの、`アド` はそこから発生・観測される個体群/個体を指す。

## 6. 基本育成・日常要素（現行）
### 6.1 時刻・睡眠
- ゲーム内時計は**現実時間と同速**で進行。
- 起動時は現実時刻に同期。
- 夜は自動睡眠状態に入る。
- 睡眠中は `Zzz` 表示と暗幕演出あり。
- `chronotype`（朝型 / 昼型 / 夜型）あり。
- `SLEEP` は**睡眠中のみ実行可能**。
- `SLEEP` 実行で `isTuckedIn = true`。
- 自然睡眠のみの場合は効果半減、寝かしつけ済みなら効果フル。

### 6.2 うんち / hygiene
- うんちは一定時間で自然発生。
- 最大3個。
- 放置で `hygiene` 低下。
- `WC` で全消去。
- 出現時にポン演出あり。
- 出現位置はモンスター左寄りへ調整済み。

### 6.3 FOOD / SLEEP / HEAL の内部処理
- `applyFood()` / `applySleep()` / `applyHeal()` 実装済み（旧 `applyFeed()` は FOOD 方針へ移行）。
- 内部主要値:
  - `hp`
  - `stamina`
  - `hunger`
  - `stability`
  - `damage`
  - `weight`
- 用語整理:
  - `mood` は現行採用しない
  - 安定度 = `stability`
  - `hunger` は内部変数名、表示上は **充足値**
- `hunger` は満腹度方式:
  - FOODで増加
  - SLEEPで減少
- FOODとHEALの役割分担:
  - FOOD内の修復系 = 軽い損傷回復や日常メンテ補助
  - HEAL = 明確な不調・深い損傷・異常状態の治療
- ログ差分は実更新値ベースで計算。
- 差分ゼロは `+0 / -0` を表示しない。
- 差分なし時は `_nochange` テンプレへ分岐する。

### 6.4 FOODメニュー（現行）
- メニュー名は **`FOOD`** を採用し、旧 `FEED` 表記は内部コード上も `FOOD` に統一済み。
- FOODは即時実行式ではなく、**所持フード一覧から1つ選んで使用**する方式。
- 画面構成は**上部選択ペイン + 下部詳細ペイン**の2段構成。
- 上部ペインは **56 / 44** 比率で、左に一覧、右にアイコンペインを置く。
- 初期開放フード:
  - `小さい肉`
  - `簡易栄養ドリンク`
- 上記2種は**無限在庫**。
- 在庫表示は `∞` を使い、`x` は表示しない。
- 右ペインは**アイコンのみ**を表示し、タイトルやフード名は出さない。
- FOODアイコンは `assets/food/meat_small.png` と `assets/food/drink_simple.png` を接続済み。
- FOOD使用後は結果表示を行い、**メインメニューへ戻らずFOOD選択画面へ戻る**。
- FOODリザルト遷移には TRN 系と同系統のフェード演出を使う。

### 6.5 HEALメニュー（現行仮実装）
- HEALは**仮実装として** `PATCH / STABILIZE / PURGE` の3メニュー構成。
- 画面構成は、左に波形、中央にAD+スロット、右に見込み、下に短文説明の4ブロック。
- 上段比率は **28 / 40 / 32**。
- 左右構成は、左をやや縮小、中央維持、右を拡張した最終比率を採用。
- 中央ADは `assets/ad/ad_media_base.png`、中央スロットは `assets/ad/ad_slot_base_v1.png` を使用。
- 中央ADは異常状態が `0` の時でもごく軽く上下する。
- 各メニューの異常状態表現:
  - `PATCH`: AD自体に大きめの欠損表現。改善で欠損が減る。
  - `STABILIZE`: ブレ + グリッチ表現。改善で収束。背面ラインは削除済み。
  - `PURGE`: 汚染 / ノイズ表現。改善で汚れが減る。汚染エフェクトは中央ペイン内に収まるよう修正済み。背面ラインは削除済み。
- 実行演出:
  - ADがスロットへ差し込まれる。
  - 差し込み時間は `220ms`。
  - 刺さった後に `120ms` 停止してから戻る。
  - フラッシュ時間は `280ms`。
  - 現行演出は背面の下地フラッシュ + 輪郭発光 + 前面白フラッシュ。
  - 十字スパーク案は削除済み。
- 差し込み中はADの上下揺れ / 横揺れを止めるが、欠損 / 汚染 / グリッチ表現は維持する。
- 右ペインは見込みゲージを大型化し、`現在値 -> 見込み値` を表示する。
- 見込み部分はゆっくり点滅する。
- `STA` も見込みゲージ表示し、コスト消費と分かる専用パターンを使う。
- 実行中は右ペインの見込み数値 / ゲージを途中で動かさず、結果はリザルトで反映する。
- `NOISE / DESYNC / CONTAMINATION` は `Lv3` から同一サイクル内で `0` まで落ちず、下限 `Lv1` を持つ。
- `DECAY` は `Lv3` から同一サイクル下限 `Lv2` を持つ。
- `damage` も重症時には同一サイクル下限あり。
- HEAL使用後は結果表示を行い、**メインメニューへ戻らずHEAL選択画面へ戻る**。
- HEALリザルト遷移には TRN 系と同系統のフェード演出を使う。

## 7. UI文字描画・メニュー（現行）
### 7.1 UI文字描画
- UI文字描画は `uiText.js` に集約済み。
- ビットマップフォント導入済み。
- 主要構成:
  - `bitmapFontSpec.js`
  - `bitmapFontData.js`
  - `bitmapTextRenderer.js`
  - `uiText.js`

### 7.2 カーソル・メニュー
- カーソル表現:
  - `▶` を使うA案を採用
  - 点滅あり
  - 移動時フラッシュあり
  - `performance.now()` ベース
- カーソル位置は**文字基準**。
- 上段メニューには左安全マージン方式を導入済み。
- メニュー枠線は上下とも削除済み。
- 線構成:
  - 上段メニュー: 下線
  - 下段メニュー: 上線
- ベースラインはフォント実寸ベースで算出。
- 下段メニュー位置は +12px 調整済み。
- ラベル群（現行採用）:
  - `STAT / FOOD / TRN / BTTL / ADV / WC / SLEEP / HEAL / EDIT`

### 7.3 UI言語共通ルール
- **予告は記号、結果は数字** を共通ルールとする。
- 増減記号の意味:
  - `+` = 小上昇
  - `++` = 中上昇
  - `+++` = 大上昇
  - `-` = 小減少
  - `--` = 中減少
  - `---` = 大減少
- 実結果は数値で表示する。
  - 例: `+2 / -1 / +0.5`
- 等級は名前ではなくUI補助情報として扱う。
- アイコン付きオブジェクトの等級表示は `I / II / III` を**右下寄りに文字描画**する。
- rank内部値:
  - `0` = なし
  - `1` = I
  - `2` = II
  - `3` = III
- 増減記号ヘルパは共通化済みで、`mid threshold = 2` / `large threshold = 3` を採用する。
- 汎用ランク表示 `I / II / III` は共通レイヤー化済みで、FOODアイコンから他カテゴリへ流用可能。
- 詳細なUI言語仕様は `DOTMON_UI言語_完全仕様.md` を参照する。

### 7.4 オーバーレイ画面と共通ヘッダー
- `STAT / FOOD / HEAL` はオーバーレイ画面として扱う。
- 表示時は直前画面のスナップショットを背面ゴーストとして薄く表示する。
- 背面ゴーストは `opacity 0.08` / `blur 1.2px` / `scale 1.01`。
- ゴースト表示ではヘッダー帯をマスクして非表示にする。
- オーバーレイ画面全体にはメイン画面と同系統の走査線を掛ける。
- 走査線は `1px線 / 3px間隔 / alpha 0.03` 系。
- 共通ヘッダーは左上にメニュー名、右上に時計を表示する。
- ヘッダー名の現行対応:
  - `status` -> `STATUS`
  - `food` -> `FOOD`
  - `heal` -> `HEAL`
  - `trnmode / trn / trnlog` -> `TRAINING`
  - `bttllog` -> `BATTLE`
- `BTTL` 本編のみはヘッダー非表示とし、`BATTLE` 表示を無理に載せない。
- `TRN` のヘッダー帯サイズは他メニューと同等へ調整済み。
- `TRN / BTTL` の暗幕フィルタは削除済みで、走査線のみ維持する。

### 7.5 デバッグメニュー
- `ESC` で開閉するデバッグメニューを実装済み。
- 表示中は本編入力を停止する。
- 構成は左カテゴリ / 右詳細 / 下説明。
- 現行カテゴリは `STATUS / ABNORMAL / FLAGS / TIME / SCREEN` 系。
- 下部文言は `HP/STB/STA/DMG を調整`、`プリセットを適用` に修正済み。

### 9.5 Page4（SKILL）
- 上部に `FINISH SKILL` と `SET 1-3` を表示する。

## 10. TRAIN（現行）
### 10.1 位置づけ
- TRNは**信号品質を整える / 上げる主成長手段**。
- 戦闘の「攻勢」軸を日常行動で整える意味を持つ。

### 10.2 基本構造
- 左: 円周波形ミニゲーム
- 右: AD表示 + 右下ミニ枠
- Aで停止判定
- Bで戻る / 退出
- Cは無効

### 10.3 モード
- `CALIBRATION`
- `BOOST SYNC`
- `NOISE CUT`

### 10.4 判定
- `BAD / NEAR / OK / SUCCESS`
- 内部成功率（安定度・充足値・スタミナ依存）とプレイヤー入力の二層構造。

### 10.5 波形・成功帯
- 高速ループ型の円周波形。
- 進行は easeInOutSine 型。
- 成功帯はセッション開始時ランダム固定。
- 成功帯は内側寄せ済み。
- 成功帯 / クリ帯突入フィードバックあり。

### 10.6 演出
- BAD時は画面全体横ブレ + 残像。
- 右ペイン瞬間フィードバックあり。
- 判定行は擬似太字化。
- トレーニング終了後は即ログ遷移せず、Aで次セッション / BでRESULTへ。
- 旧フリーズ要因だった `BTTL_SPRITE_SCALE` 未定義参照は修正済み。

### 10.7 RESULT
- 実行回数
- 信号品質 / スタミナ / 安定度の総和
- 総括フレーバー1行

### 10.8 スタミナ制御
- スタミナ0でトレーニング不可。
- 開始不可時は画面全体ブレ。
- スタミナ0到達時は強制RESULT。

### 10.9 成長保存
- トレーニング由来の成長処理・履歴保存は導入済み。
- 主成長対象:
  - `signalQuality`
  - `stability`
  - `adIntegrity`
  - `personality.*`
- 補助対象:
  - `maxHp`
  - `staminaMax`
  - `weight`
- 履歴系:
  - `trainingCount`
  - `trainingTypeCounts` または同等構造

### 10.10 効果予告表現
- TRN右下ペインの効果予告は**記号表示**を使う。
- 例:
  - `SIG ++`
  - `STA --`
- これはTRN専用ではなく、DOTMON全体の増減予告記号ルールに従う。

## 11. BTTL（現行）
### 11.1 全体
- **リアルタイムオートバトル**。
- 旧ターン制は撤廃済み。
- Bで戦闘中断し `ABORT` としてRESULTへ遷移可能。
- 戦闘開始時フリーズは修正済み。

### 11.2 画面構成
- 3ペイン構成:
  - 左: メイン戦闘
  - 右: `SIGNAL MENU` + `OBS LOG`
  - 下: 日本語ログ（意味づけ層）
- 右は事実 / 端末ログ、下は意味 / 観測記録の二層。
- BTTL本編はヘッダー非表示で、戦闘画面サイズを優先する。

### 11.3 レンジ
- 相対位置ベースで `SHORT / MID / LONG` を判定。
- 中央レンジバー上に敵味方マーカーを表示。
- `RANGE SHORT/MID/LONG` を OBS LOG に出す。
- SHORT攻撃後は反動で距離が戻る。

### 11.4 攻撃表現
- MID/LONG: 飛翔体
- SHORT: 近接寄り斬撃エフェクト（ゲート直発生）
- 着弾判定ゲート表示あり

### 11.5 HEAVY / リアクション
- `C:REACT`
- 判定:
  - BAD = 被弾
  - NEAR = DODGE
  - OK = PARRY
  - SUCCESS = REFLECT
- 反応窓中は減光 / 疑似ズーム / スロー演出あり。

### 11.6 BREAK
- 敵味方とも BREAK 値あり。
- BREAKは第2HPではなく崩れ蓄積。
- BREAKゲージ表示あり。
- 高蓄積強調あり。
- HEAVY / PARRY / REFLECT / SHORT圧などで増加。
- 自然回復あり。
- 敵BREAKゲージは下方向へ調整済み。
- 敵スプライトはやや上方向へ調整済み。
- 敵側ドライブバッジ上げ / 自側シグナルバッジ下げ方向で位置調整済み。

### 11.7 HP / STA / BREAK
- HP: 生存値
- STA: SIGNALコスト用
- BREAK: 崩れ蓄積

### 11.8 SIGNAL
- `SIGNAL MENU` から方針介入を行う。
- 現行分類:
  - `BOOST`
  - `STABILIZE`
  - `CALIBRATE`
  - `OVER CLOCK`
- SIGNALはプレイヤー側専用で、敵は同一メニューを使わない。

### 11.9 DRIVE
- 敵専用の内部駆動カテゴリとして **`DRIVE`** を採用。
- 現行分類:
  - `SURGE`
  - `GUARD`
  - `FOCUS`
  - `RAMPAGE`
- DRIVEは敵が自律的に発動する内部状態として扱う。
- プレイヤー側に対しては、敵側面などへ発動文言をそのまま表示して認識可能にする。

### 11.10 スキル制
- バトルの基盤レンジは `SHORT / MID / LONG` を維持する。
- 攻撃はレンジ抽象攻撃ではなく、**スキルベース**で行う。
- 現在レンジに対応した攻撃スキルが無い場合、そのレンジでは攻撃できない。
- ただし、以下の基礎行動は可能。
  - 接近
  - 離脱
  - 防御
- 攻撃不能時の基礎行動優先は固定ではなく、SIGNALやAI傾向で変動してよい。

### 11.11 スキル構造
- 各アドは固有スキルを **1つ** 持つ。
- 固有スキルは**必殺技枠**として扱う。
- 各アドは共有スキルを複数習得可能。
- 習得数上限は設けない。
- ただし、戦闘中に持ち込める共有スキルは **3つまで**。
- 共有スキルカテゴリは
  - 攻撃系
  - 補助系
  とする。

### 11.12 必殺技
- 固有必殺スキルは、常時使用可能ではなく**必殺技ゲージ**で解禁される。
- ゲージ蓄積は**基本、時間経過**。
- 満了時はゲージ自体が強めに点滅する。
- ゲージ上の余計な文字表示と常時 `FINISH OK` 文字表示は削除済み。
- `FINISH OK` はバッジ表示に周期表示する。
- 一戦中の使用回数は基本1回。
- 使用後は再蓄積しない。
- 発動は**プレイヤー任意**。
- 入力は**専用ボタン短押し**。
- 発動時に SIGNAL と同系統のミニゲームを実行し、結果で効果増減。
- 固有必殺にもレンジ制限を持たせる。
- 必殺ゲージ満了時は、AIが対応レンジへ寄る挙動を持つ。
- `FIN SKILL / FIN RESULT` はスキル名を出さない固定ラベルとする。
- `FIN RESULT` の結果表示は右側サークル中央へ出す。

## 13. ADV（現行仮実装）
### 13.1 基本位置づけ
- `ADV` は現状、**ノードベース探索の実プレイ土台**まで実装済みの仮実装段階とする。
- 完成系ではなく、**区画選択 / 盤面進行 / 知識進行 / ノード即効効果 / 戦闘・非戦闘分岐**の骨組み確認を目的とした段階である。
- ここで探索しているのは、実施設そのものではなく、**実施設区画に対応する観測・管理・再構成領域**である。
- ADV文言は「観測接続」「再構成領域」寄りへ調整済み。
- 現行の主軸は、**ノード内専用ミニゲームではなく、盤面を進みながら区画理解と報酬選択を行うこと**に置く。

### 13.2 ノード管理
- ADVは**ノード単位**で管理する。
- 現行ノード定義では、最低限以下を保持できる。
  - `layer`
  - `radial`
  - `stateLabels`
  - `localMapId`
  - `visibleFromStart`
  - `unlocked`
  - `specialResponseToAd`
- **全体断面図のノード1つ = 局所図1枚**を前提とする。
- 盤面探索では、区画ごとに**総ノード数おおむね10〜14、1ラン通過ノード数おおむね6〜9、最奥にボスノード1つ**の構成を採る。
- 各段の分岐は基本2分岐を中心としつつ、盤面生成時に `1〜4` 候補が出る構成を許容する。

### 13.3 現行区画構成
- 現在の可視区画は合計5つ。
- 初期探索可能区画:
  - 保守区画
  - 記録区画
  - 回線区画
- 初期可視・未解放区画:
  - 制御区画
  - 生態区画

### 13.4 区画選択画面（現行仮実装）
- 左ペインに区画一覧を表示する。
- 右ペインには**区画シンボルの大画像**を表示する。
- 下ペインには短い区画説明と `危険度 / 規模 / BOSS` を表示する。
- 区画名は理解進行に応じて `??? -> 正式名` へ変化する前提だが、初期段階では `???` を基本とする。
- 未解放区画は `LOCK` 表示とし、選択時は右ペインに薄いノイズ表現を掛ける。
- 右ペインでは**盤面縮小図を表示しない**。区画選択画面は「行き先を決める画面」として扱う。
- 区画シンボルは PNG アセットをそのまま `drawImage` で表示する。再描画はしない。

### 13.5 区画シンボル（現行）
- 区画シンボルは ADV 専用アセットとして管理する。
- 保存先:
  - `assets/adv/zone_symbols_large`
- 対応:
  - 保守区画 → `adv_zone_symbol_maintenance.png`
  - 記録区画 → `adv_zone_symbol_record.png`
  - 回線区画 → `adv_zone_symbol_circuit.png`
  - 制御区画 → `adv_zone_symbol_control.png`
  - 生態区画 → `adv_zone_symbol_ecology.png`
- 意味づけ:
  - 保守区画 = スパナ
  - 記録区画 = ファイル
  - 回線区画 = 分岐ケーブル
  - 制御区画 = パソコン / 制御端末
  - 生態区画 = 培養槽
- 画像は**再描画しない**。縦横比固定・ `imageSmoothingEnabled = false` を前提とする。

### 13.6 盤面探索画面（現行仮実装）
- 盤面は**縦向き**とし、下側を現在地、上側を奥側として表現する。
- 盤面上には主に
  - 現在地ノード
  - 次に選べるノード
  を表示する。
- 通過済みノードや遠方ノードは全面可視にせず、探索感を優先する。
- 現在地ノードには、ノードアイコンではなく**現在育成中のアドのスプライト**を表示する。
- 現在地ノードは**二重枠**で表示する。
- 選択中ノードは外枠とアイコン本体を**ゆっくり点滅**させる。
- ノード背景塗りは行わず、外枠中心で構成する。
- 接続線は**二重線**を使い、選択中ノード側の接続線には**進行方向の矢印アニメ**を表示する。

### 13.7 ノード種別（現行）
- 現行の正式ノード種別は以下。
  - `OBSERVE`
  - `COLLECT`
  - `MAINTAIN`
  - `DEFENSE`
  - `CONTAMINATE`
  - `ANALYZE`
  - `ANOMALY`
  - `BOSS`
- 非戦闘系:
  - `OBSERVE`
  - `COLLECT`
  - `MAINTAIN`
  - `ANALYZE`
- 戦闘系:
  - `DEFENSE`
  - `CONTAMINATE`
  - `BOSS`
- 条件付き特殊:
  - `ANOMALY`

### 13.8 ノードアイコン（現行）
- ノードアイコンは ADV 専用アセットとして管理する。
- 保存先:
  - `assets/adv/node_icons`
- 対応:
  - 観測 → `adv_node_icon_observe.png`
  - 採取 → `adv_node_icon_collect.png`
  - 保全 → `adv_node_icon_maintain.png`
  - 防衛 → `adv_node_icon_defense.png`
  - 汚染 → `adv_node_icon_contaminate.png`
  - 解析 → `adv_node_icon_analyze.png`
  - 異常 → `adv_node_icon_anomaly.png`
  - ボス → `adv_node_icon_boss.png`
- 意味づけ:
  - 観測 = 目
  - 採取 = コンテナ
  - 保全 = 盾
  - 防衛 = 壁
  - 汚染 = 棘 / 汚染塊
  - 解析 = 波形
  - 異常 = 三角警告
  - ボス = 角付き頭部シルエット
- 画像は**再描画しない**。縦横比固定・ `imageSmoothingEnabled = false` を前提とする。

### 13.9 名称確定・知識進行（現行仮実装）
- `advState` は保存対象に含める。
- 区画理解の進行は、少なくとも以下の軸を持てる構造を前提とする。
  - `nameProgress`
  - `purposeProgress`
  - `historyProgress`
  - `stateLabelProgress`
- 現行では、知識進行の導入と結果ログ接続までは成立した扱いとする。
- 名称確定は `??? -> 正式名` へ変化する導線を持つ。
- これは簡易実装であり、将来的には
  - 管理ログ
  - 作業ログ
  - アーカイブ
  - 端末照合
  の混合トリガーへ拡張する前提。

### 13.10 区画別報酬傾向（現行）
- 区画ごとに主報酬傾向を分離済み。
- 現行傾向:
  - 保守区画：`HEAL / MATERIAL` 寄り
  - 記録区画：`ARCHIVE` 寄り
  - 回線区画：`SIGNAL` 寄り
  - 制御区画：`control_core_fragment / archive_fragment / signal_modulator`
  - 生態区画：`bio_sample_tag / purge_filter_i / material_plate / archive_fragment`
- `汚染 / 異常 / 危険` の状態ラベルにより、副傾向差分を持てる。

### 13.11 ラン内効果保持（現行）
- ADV セッション中のみ有効な一時状態として `runEffects` を持つ。
- 現行項目:
  - `observeForecast`
  - `bossProgress`
  - `unlockedNodeIds`
  - `logs`
  - `nextBattleDamageReduction`
  - `anomalyBuff`
  - `pendingContaminateRisk`
  - `zoneCleared`

### 13.12 非戦闘ノード即効効果（現行）
- `OBSERVE`
  - 現在選択可能な各次ノードについて、**さらに次段の分岐数**を `observeForecast` に保存する。
- `COLLECT`
  - 物資を**1個確定 + 2個目確率**で取得する。
- `MAINTAIN`
  - `HP +1 / STA +2 / STB +1` の即時回復を行う。
- `ANALYZE`
  - `unlockedNodeIds` を更新して封鎖解除候補を1つ開放する。
  - 同時に `bossProgress +1` を進める。
- これらはノード選択後、そのまま `ADV RESULT` へ流して確認できる。

### 13.13 戦闘ノード即効効果（現行）
- `DEFENSE`
  - バトル勝利時、**次のバトル1回だけ**有効な被ダメ軽減を設定する。
  - 実ダメージ軽減は被ダメ計算側へ接続済み。
- `CONTAMINATE`
  - 高品質寄りの報酬を与える。
  - 追加で確率で `HP減少` または `STB低下` を適用する。
  - 適用後 `pendingContaminateRisk` は消費してクリアする。
- `ANOMALY`
  - 現行の実効果は **次の非戦闘報酬 +1**。
  - `consumeAdvAnomalyRewardBonus()` で1回消費する。
- `BOSS`
  - 勝利時に `zoneCleared = true` とし、区画選択へ戻す。
- 戦闘後反映は ADV 由来戦闘の時だけ走るよう接続済み。

### 13.14 OBSERVE の盤面予兆表示（現行）
- `observeForecast` を盤面描画で参照し、現在選択可能な次ノードの上に**短い予兆線**を表示する。
- 予兆線は
  - 1分岐 = 1本
  - 2分岐 = 2本
  - 3分岐以上 = 3本
  で表示する。
- 表示は最小記号に留め、ノード本体や種別・状態ラベルまでは見せない。
- 現行固定値:
  - 線長 `6px`
  - 2本時 `±4px`
  - 3本時 `±6px`
  - ノード上端から `6px / 12px`
  - 線幅 `1px`

### 13.15 ADV RESULT / ログ（現行）
- 非戦闘ノード効果や戦闘後反映結果は、ADV RESULT や短文ログで確認できる。
- 現行で使う短文例:
  - `先行分岐反応を検出`
  - `上位接続数を記録`
  - `資源回収を実行`
  - `状態を安定化`
  - `封鎖解除`
  - `中枢照合進行`
- 文体は短文・観測寄りを維持する。
